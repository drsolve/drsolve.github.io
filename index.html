Dixon Resultant & Polynomial System Solver

A C implementation for computing Dixon resultants and solving polynomial systems over finite fields, based on the FLINT and PML library.

GPLv2+ FLINT 3.4.0+ C Finite Fields

Features

  • Dixon resultant computation — variable elimination for multivariate polynomial systems
  • Polynomial system solver — solves n×n systems, enumerates all solutions over the given field
  • Ideal reduction mode — reduces the system via triangular ideal generators before elimination
  • Prime fields Fp with p < 263, using FLINT's modular arithmetic
  • Extension fields Fpk such as GF(28), GF(35), with custom or default irreducible polynomials
  • PML acceleration (optional) for polynomial matrix operations on prime field systems
  • File input mode for scripted and batch computation
  • Silent mode — suppresses console output while still writing results to file

Output

Every computation automatically writes a timestamped .dat solution file containing field info, input polynomials, computation time, and results.

  • Command-line input → solution_YYYYMMDD_HHMMSS.dat
  • File input example.dat → example_solution_YYYYMMDD_HHMMSS.dat

Usage

All modes share the same binary: ./dixon. Choose a mode below.

Dixon Resultant (Basic)

Eliminates variables from a polynomial system and returns the resultant in the remaining variables.

Syntax

./dixon "polynomials" "eliminate_vars" field_size
The number of variables to eliminate must equal the number of equations minus one.

Example — eliminate x and y from a 3-polynomial system over GF(257):

./dixon "x+y+z, x*y+y*z+z*x, x*y*z+1" "x,y" 257

Output: the resultant polynomial in z, written to a .dat file.

Polynomial System Solver

Solves a well-determined system of n equations in n unknowns and enumerates all solutions over the given field.

Syntax

./dixon --solve "polynomials" field_size

Example

./dixon --solve "x^2 + y^2 + z^2 - 6, x + y + z - 4, x*y*z - x - 1" 257

Variables are auto-detected from the expressions. Use --silent to suppress console output while still writing the solution file:

./dixon --silent --solve "..." 257

Extension Fields

Specify the field as p^k. The generator variable is t by default. FLINT's built-in irreducible polynomial is used unless a custom one is provided.

Default polynomial

./dixon "x + y^2 + t, x*y + t*y + 1" "x" 2^8

Custom polynomial (e.g., the AES field polynomial for GF(28)):

./dixon --solve "x^2 + t*y, x*y + t^2" "2^8: t^8 + t^4 + t^3 + t + 1"
FieldArgument
GF(28) default2^8
GF(28) AES polynomial"2^8: t^8+t^4+t^3+t+1"
GF(35) default3^5
GF(35) custom"3^5: t^5+2*t+1"

Dixon with Ideal Reduction

Reduces the polynomial system modulo a triangular ideal before computing the Dixon resultant. Useful for structured systems such as those arising from block cipher analysis.

Syntax

./dixon "polynomials" "eliminate_vars" "ideal_generators" field_size

Example

./dixon \
  "a1^2 + a2^2 + a3^2 + a4^2 - 10, a4^3 - a1 - a2*a3 - 5" \
  "a4" \
  "a2^3 = 2*a1 + 1, a3^3 = a1*a2 + 3, a4^3 = a1 + a2*a3 + 5" \
  257

File Input

Pass a .dat file instead of inline arguments for scripted or batch use.

./dixon example.dat
./dixon --solve system.dat

Dixon mode — file format

257            # line 1: field size
x+y+z          # line 2+: polynomials (one per line or comma-separated)
x*y+y*z+z*x
x*y*z+1
x,y            # last line: variables to eliminate

Solver mode — file format

257                   # line 1: field size
x^2 + y^2 + z^2 - 6  # line 2+: n equations in n variables
x + y + z - 4
x*y*z - x - 1

Output is written to example_solution_YYYYMMDD_HHMMSS.dat.

Finite Field Support

DixonRes supports prime fields and extension fields. All arithmetic uses FLINT, with optional PML acceleration for prime field systems.

Prime fields — Fp

Any prime p with p < 263 is supported. Specify the field as a plain integer.

./dixon --solve "..." 257
./dixon --solve "..." 65537
./dixon --solve "..." 4294967311

When the optional PML library is installed and detected at configure time, polynomial matrix operations are automatically accelerated for well-determined prime field systems.

Extension fields — Fpk

Specified as p^k. The generator is the variable t, satisfying a chosen irreducible polynomial over Fp. FLINT's built-in default is used if no polynomial is provided.

# FLINT default polynomial
./dixon "..." 2^8

# Custom polynomial
./dixon "..." "2^8: t^8 + t^4 + t^3 + t + 1"
Extension fields are slower than prime fields due to polynomial arithmetic overhead. PML acceleration does not apply to extension field computations.

Installation

DixonRes requires FLINT 3.4.0 or later. PML is optional but recommended for prime field performance.

1. Install FLINT

Get FLINT 3.4.0 or newer from github.com/flintlib/flint and follow its build instructions for your platform.

2. Install PML (optional)

Get PML from github.com/vneiger/pml. If present, DixonRes will detect and use it automatically at configure time for prime field systems.

3. Clone and build

git clone https://github.com/DixonRes/DixonRes
cd DixonRes
./configure
make
make install   # optional

For additional build options:

./configure --help
make help

4. Verify

Run a quick test to confirm the build is working:

./dixon --solve "x + y - 3, x*y - 2" 257

Expected: solutions [x=1, y=2] and [x=2, y=1] over GF(257).
